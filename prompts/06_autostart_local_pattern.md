# 프롬프트 #6 — 자동시작 + 로컬 부트스트랩 패턴 (⭐ 부팅 지연 디버깅)

## 클로드한테 던진 원문

> 자비스 부팅하면 자동 시작되게 하고 싶어. Startup 폴더에 vbs 넣었더니 부팅이 30초씩 느려졌어. 왜 그런지 디버깅하고 부팅 지연 없는 패턴으로 바꿔줘.

## 받은 결과 요약

**원인 분석**:
- Startup 폴더의 `.lnk`/`.vbs`는 윈도우 셸이 **로그인 시 동기적으로** 실행
- vbs가 클라우드 드라이브 경로(`G:\내 드라이브\...`)를 직접 가리키면 → **드라이브 마운트 대기로 셸 블록**
- 로그인 → 데스크탑 표시 사이에 G드라이브 인증·마운트가 끼면서 30~60초 지연

**해결 — 로컬 부트스트랩 패턴**:

```
Startup\Jarvis_Voice.lnk
     ↓
%LOCALAPPDATA%\voice_assistant\bootstrap.vbs   ← 로컬!
     ↓ WScript.Sleep 45000  (로그인 후 45초 대기)
     ↓ Do While Not fso.FileExists(mainPy) ...  (마운트 폴링, 5초 간격, 최대 5분)
     ↓ pythonw 동적 탐색 (Python 3.10~3.13 후보 12개)
     ↓ ws.Run "pythonw.exe main.py", 0, False  (백그라운드)
```

핵심: **Startup이 가리키는 파일은 항상 로컬**. 메인 스크립트(클라우드)는 그 안에서 폴링으로 기다림.

## 실측 트러블 + 해결

| 증상 | 원인 | 해결 프롬프트 |
|------|------|--------------|
| 로그인 직후 vbs 실행 → 디스크 경합으로 자비스 자체도 느림 | 부팅 직후 윈도우 인덱서·OneDrive·백그라운드 작업 폭주 | "WScript.Sleep 45000 추가해서 시스템 안정화 후 시작" |
| 클라우드 드라이브 마운트 전 main.py 못 찾고 종료 | 즉시 fso.FileExists 검사 | "FileExists Loop로 5초 간격 폴링, 최대 5분 대기" |
| 클라우드 미마운트 PC에서 매 부팅 5분 대기 | 폴링이 무조건 끝까지 | "5분 timeout 후 조용히 WScript.Quit 0 — 다음 부팅에 재시도" |
| Python 경로가 PC마다 다름 | 사용자가 어디 설치했는지 모름 | "Python 3.10~3.13 후보 12개 미리 박고 첫 매치 사용, 없으면 PATH 폴백" |

→ 최종 결과물: `bootstrap.vbs` (이 리포 루트). `__MAIN_PY__`/`__WORK_DIR__` 토큰은 install.bat이 치환.
